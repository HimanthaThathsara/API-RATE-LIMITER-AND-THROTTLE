# Contributing

Thanks for your interest in contributing! We welcome contributions of all
shapes and sizes. Please follow these guidelines to make the process smooth for
everyone.

## Code of Conduct

Be kind and respectful. This project follows a code of conduct — contributors
are expected to be welcoming and professional.

## Getting started

1. Fork the repository and clone your fork.
2. Create a feature branch: `git checkout -b feat/your-feature`.
3. Install dependencies: `npm install`.
4. Run tests and linters before committing: `npm test` and `npm run lint`.

If you're planning a large or breaking change, open an issue first to discuss
the approach with the maintainers.

## Commit messages

Keep commits small and focused. Use imperative, present-tense messages, e.g.
`Add token-bucket algorithm`.

## Branching & PRs

- Base new work from `main`.
- Include tests for bug fixes and new features.
- Describe the motivation and changes in the PR description and link any
  related issues.

## Tests

- Add unit tests for new behaviors. Run `npm test` locally.
- Keep tests deterministic and fast.

## Style

Follow the project's ESLint / Prettier configuration. If a config file is not
present, follow common JavaScript style — 2-space indentation, semicolons
optional, single quotes for strings.

## Security issues

Do not open public issues about security vulnerabilities. Contact the
maintainers privately (email or the project's security contact) with details so
we can triage and fix before public disclosure.

## Thank you

We appreciate your help improving this project. If you're new to open source
and want guidance, start with small issues and we'll help you along the way.
